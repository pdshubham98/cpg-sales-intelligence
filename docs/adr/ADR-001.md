# ADR-001: Technology Stack Decisions

**Status:** Accepted  
**Date:** 2026-06-19  
**Author:** Pradeep Kesharwani

---

## Context

We are building a lightweight, production-style AI-powered CPG sales analytics platform.
The rubric rewards completeness and correctness over sophistication. Every component must
run end-to-end within a single Docker container. The solution must be explainable and
demonstrable in a 7-minute video.

---

## Decisions

### 1. Python 3.11

Python is the lingua franca of data engineering and ML. All tooling in this stack has
first-class Python support. No secondary language is required.

### 2. FastAPI over Flask / Django

FastAPI provides automatic OpenAPI documentation, async support, and Pydantic validation
out of the box. Flask requires manual schema validation; Django is overweight for a REST API.
FastAPI's `lifespan` context manager cleanly handles startup ingestion without a separate
script.

### 3. SQLite over PostgreSQL

The rubric evaluates "a Dockerfile that runs." PostgreSQL adds a second container, health
checks, init scripts, and volume management. SQLite is a file — zero ops overhead, works
identically locally and inside Docker, and supports WAL mode for concurrent reads.
Extension to PostgreSQL requires changing only `get_connection()` in `schema.py`.

### 4. Pandas over PySpark

The dataset is ~200 rows. PySpark requires a JVM, adds 500 MB to the Docker image, and
takes 10–15 seconds to start a SparkContext — all for data that fits in RAM thousands of
times over. In a production environment with billions of rows, the `loader.py` ingestion
module is the only file that would change; the PySpark API mirrors pandas for this pattern.
This trade-off is documented and mentioned explicitly in the video demo.

### 5. Groq (primary) + Gemini (fallback) over OpenAI / Anthropic

| Provider | Reason |
|---|---|
| Groq | Free tier, no credit card, 14,400 req/day, llama-3.3-70b quality |
| Gemini | Free 1M tokens/day (or paid account available as backup) |
| OpenAI | Requires paid account — excluded by constraint |
| Anthropic | Excluded by constraint (different work context) |

Provider is controlled by a single `LLM_PROVIDER` env var. Swapping requires zero code
changes.

### 6. scikit-learn LinearRegression over Prophet / XGBoost

Linear regression on monthly-aggregated time series is sufficient for a 3–12 month
horizon on ~15 data points per dimension. It is interpretable, requires no GPU, and has
no warm-up time. Cross-validated R² is reported alongside predictions for transparency.
Prophet or XGBoost are the natural upgrades (documented in `architecture.md`).

### 7. Streamlit over React

The rubric requires a working UI, not a production frontend. Streamlit delivers a
multi-page dashboard with charts in ~200 lines of Python, no build step, no Node.js, and
no CORS configuration. Plotly provides production-quality charts. Business users can use
it without installing anything.

### 8. GitHub Actions over GitLab CI

The project repository is hosted on GitHub (`pdshubham98/cpg-sales-intelligence`).
GitHub Actions is the native CI solution — no separate runner setup required. The pipeline
has three stages: lint (flake8), test (pytest with coverage), and Docker build + smoke
test (main branch only).

---

## Consequences

- The stack is deployable on any machine with Docker installed — no cloud account required
- Tests run in 2 seconds with no live API keys
- Swapping any component (storage, LLM, model) requires changing one file and one env var
- PySpark and PostgreSQL are documented as the natural production upgrade path
